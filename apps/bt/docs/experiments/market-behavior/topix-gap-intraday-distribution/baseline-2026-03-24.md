## 解釈

### 1. `<= -2σ` は明確な intraday rebound

- `gap <= -1.35%` では全 group がプラスで、`TOPIX100` / `TOPIX500` は `0.26%` 台まで伸びる。
- fixed `-2%` よりサンプルが十分増えたため、`極端な下窓での反発` を実用寄りの件数で見られるようになった。
- 読み: weak regime は依然として有効で、主戦場は大型株と TOPIX500。

### 2. `-1σ ~ +1σ` は「TOPIX100 / TOPIX500 ほぼ flat、PRIME 系はやや弱い」

- 最頻出の中立帯では `TOPIX100` / `TOPIX500` はほぼゼロ、`PRIME` / `PRIME ex TOPIX500` は小幅マイナス。
- 読み: broad universe を毎回触るより、中立帯を見送る設計の方が筋がいい。

### 3. `+1σ ~ +2σ` は small/mid 側に rotation

- `0.68% <= gap < 1.35%` では `PRIME` / `PRIME ex TOPIX500` がプラス、`TOPIX100` / `TOPIX500` はマイナス。
- 読み: moderate positive gap は大型の押し目買いではなく、やや広めの risk-on rotation で解釈した方が合う。

### 4. `>= +2σ` でも strong side はまだ死なない

- `gap >= 1.35%` では全 group がプラスで、特に `PRIME ex TOPIX500` が `0.2094%` と強い。
- 読み: 上側 extreme は fixed `2%` のときほど sparse ではなく、follow-through として扱える水準が残っている。

### 5. 頻度込みでは `TOPIX100` / `TOPIX500` がわずかに勝つ

- `TOPIX100` と `TOPIX500` は weak bucket の正寄与と中立帯の耐久でプラスに残る。
- `PRIME ex TOPIX500` は strong 側は強いが、中立帯と moderate negative 帯の負けが重い。
- 読み: gap 条件だけで intraday を回すなら、default 候補は `TOPIX100` / `TOPIX500`。

## 簡易 Rotation ルール

ルール:

- `weak => TOPIX500 long`

## Rotation の解釈

### 1. sigma 化で `strong` 側も主力になった

- fixed `1%/2%` 時代は `weak` 側の寄与が優勢だったが、`±0.675%` 基準にすると `strong` 側の trade day が 396 日まで増え、平均 return も `weak` を上回った。
- 読み: gap の絶対幅ではなく、その日の平常スケールに対する相対的な強さで切ると、strong side の使い道が見えてくる。

### 2. neutral flat の合理性は依然として高い

- 70.56% を占める中立帯は、group selection を工夫しても大きな期待値は出にくい。
- 読み: この rotation の本質は「gap がない日は入らない」にある。

### 3. 取引頻度は上がったが drawdown も浅くはない

- trade days は 723 日まで増え、累積 75.77% と十分な成績になった。
- 一方で max drawdown は `-11.24%` なので、signal の単純適用だけで過信はできない。
- 読み: 実運用前には期間分割、コスト、執行スリッページを必ず入れるべき。

## 注意点

- intraday は `open -> close` のみで、寄り付き約定スリッページや実際の執行コストは含まない。
- equal-weight の群平均であり、現実の売買可能ポートフォリオそのものではない。
- 今回の bucket は `0%` を基準に `±σ` 分割している。平均値は併記しているが、bucket center には使っていない。

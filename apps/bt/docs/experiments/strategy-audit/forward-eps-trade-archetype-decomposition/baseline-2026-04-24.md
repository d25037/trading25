## Prime Ex TOPIX500 Overheated Tail

`primeExTopix500_20260325` の full-history は 284 trades、avg +14.00%、median -3.52%、win rate 40.85%、severe loss rate 22.18%、worst -35.34%。

Value / size の追加 overlay だけでは、実運用で採用しやすい安定改善にはなりにくかった。一方で worst 側を見ると、悪いサブセットは「直近リターン過熱」と「risk-adjusted return 高位」が重なった銘柄に集中していた。

Q5 threshold:

- `stock_return_60d_pct >= 58.78`
- `stock_return_20d_pct >= 33.71`
- `risk_adjusted_return_value >= 3.886`

Correlation / overlap:

- 連続値の Pearson correlation は、20d vs 60d が 0.621、60d vs risk-adjusted が 0.635、20d vs risk-adjusted が 0.444。
- Spearman correlation は、20d vs 60d が 0.470、60d vs risk-adjusted が 0.733、20d vs risk-adjusted が 0.516。
- Q5 flag の phi correlation は、20d/60d が 0.503、60d/risk-adjusted が 0.481、20d/risk-adjusted が 0.298。
- 3条件すべて Q5 の銘柄は 23 trades、avg -4.56%、median -17.49%、severe loss rate 65.22%。
- Q5 条件が2つ以上重なる銘柄は 46 trades、avg -3.72%、median -8.90%、severe loss rate 45.65% で、悪い尾の中心だった。
- `risk_adjusted_return_value` Q5 単独は avg +24.81%、`risk_q5_only` は avg +69.87% の右尾を含むため、単独除外は期待値を削る。

Candidate exclusion:

```yaml
exclude_if:
  overheated_overlap_count:
    conditions:
      - stock_return_60d_pct >= 58.78
      - stock_return_20d_pct >= 33.71
      - risk_adjusted_return_value >= 3.886
    min_count: 2
```

この `min_count >= 2` 除外を入れると、kept 238 trades、avg +17.42%、median -2.15%、win rate 44.96%、severe loss rate 17.65%、worst -22.70%。baseline 比で avg +3.42pt、median +1.37pt、severe loss rate -4.54pt になった。

Interpretation: `forward_eps_driven` の改善は「低PBR・低PER・小型の良いサブセットを探す」より、「過熱が複数軸で重なる悪い尾を除く」方が先に検討すべき。特に `risk_adjusted_return` は良い右尾も持つため、単独で Q5 を切るのではなく、20d/60d の短期過熱と重なった場合だけ除外するのが妥当。

## Decision

- 先行研究と違う点は、対象が annual all-stock panel ではなく、`forward_eps_driven` の実約定母集団に限定されていること。
- Prime は `small_market_cap_q1` を追加候補にする価値があるが、capacity と右尾依存を別途確認する。
- Standard は full-history の value overlay が強いが、holdout で崩れているため即採用しない。
- `ADV60 >= 10mn` は主要選択条件にしない。結果表には ADV60 中央値を残し、流動性・capacity の診断に使う。
- Prime ex TOPIX500 では、過熱3条件のうち2つ以上が重なる銘柄を除外する `overheated_overlap_count` が最優先の実装候補。これは現時点では研究結果であり、production YAML の実行条件としてはまだ有効化していない。
